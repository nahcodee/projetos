# Prática de Gitflow e Versionamento

Repositório criado para a disciplina de Versionamento de Código, com o objetivo de praticar o fluxo de trabalho **Gitflow** utilizando o Git Bash e o GitHub.

##  Sobre o Projeto
Este projeto consiste na simulação de um fluxo de desenvolvimento profissional. O foco principal não é o código em si, mas a estrutura de ramificação (branching) e a organização dos commits.

## Metodologia Aplicada (Gitflow)
Para este exercício, segui o padrão Gitflow ensinado no workshop, utilizando a seguinte estrutura de branches:

1.  **main**: Branch de produção (código estável).
2.  **develop**: Branch base para o desenvolvimento.
3.  **feature/**: Branches temporárias criadas a partir da `develop` para adicionar novas modificações (ex: `feature/minha-primeira-tarefa`).

### Passos realizados:
- Inicialização do repositório local e remoto.
- Criação e switch para a branch `develop`.
- Criação de uma branch de `feature` para simular uma tarefa.
- Merge da `feature` de volta para a `develop`.
- Abertura de **Pull Request** no GitHub para levar as alterações da `develop` para a `main`.

##  Ferramentas Utilizadas
- **Git Bash** (Terminal)
- **GitHub** (Repositório Remoto)
- **VS Code / Bloco de Notas** (Edição de arquivos)
